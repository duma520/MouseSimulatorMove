# 1. 前言：为什么需要鼠标抖动模拟器

## 1.1 软件定位

**鼠标抖动模拟器**是一款轻量级、开源、免费的Windows桌面工具，通过定时微小幅度移动鼠标并立即复位，达到**模拟用户活动**、**防止系统休眠**、**保持在线状态**的目的。

## 1.2 核心价值

在当今数字化工作环境中，我们经常面临以下困扰：

| 问题 | 后果 | 本软件解决方案 |
|------|------|---------------|
| 公司电脑自动锁屏 | 频繁输入密码，打断工作流 | 每60秒移动1像素，保持系统活跃 |
| 远程会议中屏幕熄灭 | 无法随时查看资料 | 后台微动，屏幕保持常亮 |
| 长时间下载/上传 | 系统休眠导致中断 | 定时唤醒系统，保持网络连接 |
| 网页/IM状态显示离线 | 错过重要消息 | 模拟用户活动，保持"在线"状态 |
| 演示大屏自动关闭 | 影响展示效果 | 防止屏幕保护程序启动 |

## 1.3 设计理念

- **极简**：打开即用，无需复杂配置
- **隐形**：1像素移动，肉眼无法察觉
- **可靠**：多线程设计，永不卡顿
- **开放**：完全开源，可自行修改编译

# 5. 进阶指南：高级功能详解

## 5.1 参数设置深度解析

### 移动间隔（5-3600秒）

**参数意义**：两次鼠标抖动之间的等待时间。

**设置建议：**

| 使用场景 | 推荐间隔 | 原因 |
|---------|---------|------|
| 公司防锁屏 | 60-120秒 | 大多数公司锁屏策略为5-10分钟 |
| 保持IM在线 | 300秒（5分钟） | 微信/QQ离线判断通常为10分钟 |
| 防休眠下载 | 30-60秒 | 确保系统完全不会休眠 |
| 演示/会议 | 180秒（3分钟） | 屏幕保护程序通常5分钟后启动 |

**高级技巧**：如果您不确定公司的锁屏策略，可以先设为30秒观察一段时间，如果还是锁屏，再缩短间隔。

### 移动方向（8种选择 + 随机）

**参数意义**：鼠标抖动的方向。

**方向详解：**

```
     ↖ ↑ ↗
     ← · →
     ↙ ↓ ↘
```

- **向上/向下/向左/向右**：单一方向移动，最基础的模拟
- **左上/右上/左下/右下**：对角线移动，更接近人类操作习惯
- **随机方向**：软件每次从8个方向中随机选择一个，最推荐的设置

**专业分析**：真实用户在使用鼠标时，很少做纯粹的垂直或水平移动。对角线移动（如从左上到右下）是最常见的操作轨迹。因此，2.0版本的随机方向包含了8个方向，大大增强了模拟的真实性。

### 移动距离（1-10像素）

**参数意义**：鼠标离开原位多远后返回。

**直观理解**：
- **1像素** ≈ 头发丝粗细，肉眼完全无法察觉
- **3像素** ≈ 轻微抖动，仔细观察才能发现
- **5像素** ≈ 明显跳动，可能引起注意
- **10像素** ≈ 显著移动，容易被发现

**推荐设置**：始终设为**1像素**。1像素足以触发系统事件（如重置空闲计时器），但完全不影响您的工作。

**技术原理**：Windows系统检测用户活动的机制基于"输入事件"——任何鼠标移动、键盘按键都会重置系统的空闲计时器。1像素的移动足够产生一个"鼠标移动事件"，系统会认为"用户还在"。

```

向下移动: 21

右上移动: 15
左下移动: 13

```

**统计信息的价值**：

1. **验证工作状态**：如果数字持续增长，说明软件正常工作
2. **分析随机性**：在随机模式下，8个方向的计数应大致均衡
3. **估算运行时长**：运行时间 = 总次数 × 间隔（大致）

### 实时状态监控

主界面上的状态信息提供了实时反馈：

- **最后方向**：最近一次移动的方向（带箭头图标）
- **下次移动**：预计下一次操作的时间

## 6.2 关键技术实现

### 6.2.1 鼠标移动实现

```python
# 获取当前位置
current_x, current_y = win32api.GetCursorPos()

# 计算新位置（对角线逻辑）
new_x, new_y = current_x, current_y
if "up" in dir_choice:
    new_y = current_y - self.move_distance
elif "down" in dir_choice:
    new_y = current_y + self.move_distance

if "left" in dir_choice:
    new_x = current_x - self.move_distance
elif "right" in dir_choice:
    new_x = current_x + self.move_distance

# 移动鼠标到新位置
win32api.SetCursorPos((new_x, new_y))
time.sleep(0.05)  # 短暂停留，确保系统检测到
```

# 9. 技术原理深度解析

## 9.1 Windows系统空闲检测机制

### 系统如何知道用户"空闲"？

Windows通过 **GetLastInputInfo()** 函数追踪最后输入时间：

```c
LASTINPUTINFO lii;
lii.cbSize = sizeof(LASTINPUTINFO);
GetLastInputInfo(&lii);
DWORD idleTime = GetTickCount() - lii.dwTime;
```

任何鼠标移动、键盘按键、触摸输入都会重置这个计时器。

### 屏幕保护程序触发条件

当 `idleTime` 超过屏幕保护程序设置的延迟时间（通常5-15分钟），系统会：
1. 如果启用了屏幕保护程序，启动屏保
2. 如果设置了"空闲后休眠"，进入休眠
3. 如果设置了"关闭显示器"，关闭屏幕

### 本软件如何工作

通过每隔一段时间（如60秒）执行一次1像素的鼠标移动，我们持续重置 `GetLastInputInfo()` 的计时器，使系统认为用户始终处于活动状态。

## 9.2 1像素移动的技术意义

### 为什么1像素足够？

Windows的鼠标事件处理机制中，**任何非零的鼠标移动**都会产生一个输入事件。即使只是1像素的移动，系统也会：

1. 记录输入时间
2. 更新光标位置
3. 触发可能的钩子（hook）函数
4. 重置空闲计时器

### 为什么立即返回？

立即返回是为了最小化对用户的影响。整个过程在50ms内完成：
- 移动到 (x+1, y)
- 等待50ms（确保系统捕获事件）
- 移回 (x, y)

用户几乎无法察觉。

## 9.3 多线程与界面响应性

### 事件循环与工作线程

PyQt5应用运行在主线程的事件循环中。如果我们在主线程中执行耗时的鼠标操作和等待，界面会"卡死"。

**解决方案**：使用QThread将工作移到后台线程。

```python
# 主线程
self.mouse_thread = MouseSimulatorThread()
self.mouse_thread.start()

# 工作线程
class MouseSimulatorThread(QThread):
    def run(self):
        while True:
            self.perform_move()
            self.msleep(100)  # 非阻塞等待
```

### 线程间通信

使用Qt的信号（Signal）机制实现线程安全通信：

```python
# 工作线程定义信号
update_count = pyqtSignal(int)

# 发送信号
self.update_count.emit(self.count)

# 主线程连接信号
self.mouse_thread.update_count.connect(self.update_count)
```

## 9.4 设置持久化原理

### JSON作为配置文件

选择JSON的原因：
- 人类可读/可编辑
- 跨平台兼容
- Python原生支持

### 存储位置

配置文件保存在：

### 窗口状态保存

## 9.5 高DPI适配

### 什么是DPI缩放？

高分辨率屏幕上，Windows会对应用程序进行缩放，使界面不至于太小。但缩放可能导致界面模糊或布局错乱。

### 解决方案

```python
# 设置DPI感知
try:
    ctypes.windll.shcore.SetProcessDpiAwareness(1)  # Windows 8.1+
except:
    ctypes.windll.user32.SetProcessDPIAware()      # Windows Vista/7
```

这告诉Windows："我自己处理DPI缩放，你别动我的界面"。确保控件大小正确、文字清晰。

```python
            try:
                if not self.is_paused:
                    # 获取当前位置
                    current_x, current_y = win32api.GetCursorPos()
                    
                    # 确定方向（随机或指定）
                    if self.direction == "random":
                        dir_choice = random.choice([
                            "up", "down", "left", "right",
                            "upleft", "upright", "downleft", "downright"
                        ])
                    else:
                        dir_choice = self.direction
                    
                    # 计算新位置
                    new_x, new_y = current_x, current_y
                    
                    # 处理垂直移动
                    if "up" in dir_choice:
                        new_y = current_y - self.move_distance
                    elif "down" in dir_choice:
                        new_y = current_y + self.move_distance
                    
                    # 处理水平移动
                    if "left" in dir_choice:
                        new_x = current_x - self.move_distance
                    elif "right" in dir_choice:
                        new_x = current_x + self.move_distance
                    
                    # 特殊处理纯方向（避免对角线误判）
                    if dir_choice in ["up", "down", "left", "right"]:
                        if dir_choice == "up":
                            new_y = current_y - self.move_distance
                        elif dir_choice == "down":
                            new_y = current_y + self.move_distance
                        elif dir_choice == "left":
                            new_x = current_x - self.move_distance
                        elif dir_choice == "right":
                            new_x = current_x + self.move_distance
                    
                    # 执行移动
                    win32api.SetCursorPos((new_x, new_y))
                    time.sleep(0.05)  # 短暂停留
                    win32api.SetCursorPos((current_x, current_y))
                    
                    # 更新统计
                    self.count += 1
                    self.update_count.emit(self.count)
                    
                    # 方向名称映射
                    dir_names = {
                        "up": "↑ 上", "down": "↓ 下",
                        "left": "← 左", "right": "→ 右",
                        "upleft": "↖ 左上", "upright": "↗ 右上",
                        "downleft": "↙ 左下", "downright": "↘ 右下"
                    }
                    self.update_direction.emit(dir_names.get(dir_choice, dir_choice))
                    
                    # 更新时间信息
                    current_time = datetime.now().strftime("%H:%M:%S")
                    self.update_status.emit(f"已执行操作 - {current_time}")
                    
                    self.next_move_time = time.time() + self.interval
                    next_time_str = datetime.fromtimestamp(
                        self.next_move_time).strftime("%H:%M:%S")
                    self.update_next_time.emit(next_time_str)
                
                # 等待（分段等待，便于响应停止命令）
                for _ in range(self.interval * 10):
                    if not self.is_running:
                        return
                    if self.is_paused:
                        time.sleep(0.1)
                    else:
                        time.sleep(0.1)
                        if time.time() >= self.next_move_time:
                            break
                            
            except Exception as e:
                self.update_status.emit(f"错误: {str(e)}")
                time.sleep(1)
    
    def stop(self):
        self.is_running = False
    
    def pause(self):
        self.is_paused = True
    
    def resume(self):
        self.is_paused = False
```

## 11.3 关键算法说明

### 方向处理算法

2.0版本的核心改进是方向处理逻辑。原来的代码使用简单的if-elif处理4个方向，现在需要处理8个方向，包括对角线组合。

**对角线处理逻辑**：
```python
# 使用字符串包含判断组合方向
if "up" in dir_choice:
    new_y = current_y - self.move_distance
if "left" in dir_choice:
    new_x = current_x - self.move_distance
# 同时包含up和left就是左上角
```

**纯方向特殊处理**：为了避免对角线误判，对纯方向（up/down/left/right）进行了单独处理。

```python
remaining = self.mouse_thread.next_move_time - current_time

if remaining > 0:
    progress = int((1 - remaining / total) * 100)
```
